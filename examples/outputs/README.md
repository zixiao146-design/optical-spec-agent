# Demo Outputs

Real parser output generated by the rule-based parser + validator pipeline. Each file can be reproduced by running the corresponding example script.

## How to reproduce

```bash
# Option 1: via CLI
optical-spec parse "<input text>" -o examples/outputs/demo_gap_plasmon_sweep.json

# Option 2: via example scripts
python examples/example_01_nanoparticle_gap_plasmon.py
python examples/example_02_asymmetric_gold_cross.py
python examples/example_04_comsol_mode_analysis.py
```

## Status field semantics

Every field in the spec carries a `status` tag:

- **confirmed**: Directly extracted from the input text via keyword/regex matching
- **inferred**: Not explicitly stated; derived by post-hoc inference rules (logged in `assumption_log`)
- **missing**: Not found in input and no inference rule applies; listed in `missing_fields`

The `validation_status` section reports whether the spec is executable (all required fields present) and lists specific errors and warnings.
