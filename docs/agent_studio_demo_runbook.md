# Agent Studio Demo Runbook

## 1. Purpose

A local maintainer-facing walkthrough for demonstrating Agent Studio without
cloud services, solver execution, external LLM calls, uploads, tags, or
releases.

## 2. Current Status

- Current public prerelease: v0.9.0rc8
- Current main development version: 0.9.0rc9.dev0
- Frontend MVP: implemented and hardened
- API contract version: 0.1
- PyPI: not published
- TestPyPI uploaded/verified only for 0.9.0rc6.dev0
- v0.9.0rc9 tag: not created
- v1.0.0 tag: not created

## 3. Start Demo

Recommended command:

```bash
./scripts/demo_agent_studio.sh
```

First-run quickstart command:

```bash
./scripts/bootstrap_demo_env.sh
source /tmp/osa-agent-studio-demo/bin/activate
./scripts/run_quickstart_demo.sh
```

The user-facing quickstart is documented in `docs/quickstart.md` and
`docs/quickstart.zh-CN.md`.
The Chinese step-by-step tutorial is documented in
`docs/agent_studio_chinese_guided_tutorial.md`, with terminology in
`docs/frontend_chinese_terminology.md`. Real maintainer feedback is recorded in
`docs/agent_studio_demo_feedback.md`.

The script runs the API and frontend smoke checks, starts the local API and
frontend, prints the URLs, verifies readiness, and exits by default. To keep the
demo running until Ctrl-C:

```bash
OSA_DEMO_HOLD=1 ./scripts/demo_agent_studio.sh
```

To keep it up for a fixed window:

```bash
OSA_DEMO_RUN_SECONDS=300 ./scripts/demo_agent_studio.sh
```

Optional/manual visual smoke can be included with:

```bash
OSA_DEMO_WITH_VISUAL=1 ./scripts/demo_agent_studio.sh
```

Manual alternative:

```bash
python -m uvicorn optical_spec_agent.api.app:app --reload --host 127.0.0.1 --port 8000
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

## 5. Demo Walkthrough

### Dashboard / Readiness

- What to click: open the frontend URL and select Dashboard / Readiness.
- Quickstart action: use Start guided demo to follow the eight-step local
  agent walkthrough.
- Expected result: readiness cards show `v0.9.0rc6`,
  `0.9.0rc7`, API contract version 0.1, PyPI not published, and
  recommended next actions.
- Safety boundary to point out: no solver execution by default and no
  PyPI/TestPyPI or GitHub release controls.

### Spec Input

- What to click: use Load example spec, then Parse locally or Validate JSON.
- Expected result: parsed spec or validation diagnostics render with raw JSON.
- Safety boundary to point out: demo fixture loading is not live validation
  until submitted, and no external LLM is called by default.

### Adapter Matrix

- What to click: select Adapter Matrix.
- Expected result: Gmsh, Meep, MPB, Optiland, and Elmer appear with maturity
  and evidence context.
- Safety boundary to point out: adapter discovery does not execute solvers.

### Workflow Plan

- What to click: load the workflow fixture, then generate a workflow plan.
- Expected result: local workflow plan steps and diagnostics appear.
- Safety boundary to point out: workflow planning is preview-first and no
  solver is executed.

### Artifact Preview

- What to click: choose a tool such as `gmsh`, load minimal spec, and generate
  preview.
- Expected result: preview content or artifact summary appears.
- Safety boundary to point out: preview artifacts are not production-grade
  physical validation.

### Validation Evidence

- What to click: select Validation Evidence.
- Expected result: Gmsh, Meep, MPB, and Optiland show Level 3; Elmer remains
  Level 2 + Level-3-ready, install deferred.
- Safety boundary to point out: formal convergence proof is not claimed.

### System Status

- What to click: select System Status.
- Expected result: health, version, schema metadata, and API base URL render.
- Safety boundary to point out: the UI does not control package uploads, tags,
  or GitHub releases.

## 6. Safety Boundaries

- No solver execution by default.
- No external LLM call by default.
- No PyPI/TestPyPI upload controls.
- No tag/release controls.
- No production-grade validation claim.
- No formal convergence proof claim.
- Demo fixture mode is clearly labeled and is not live validation.

## 7. Stop Demo

Press Ctrl-C if `OSA_DEMO_HOLD=1` is used. The script uses trap cleanup for the
local API/frontend background processes and removes frontend build/report
artifacts generated by the demo script.
