# Validation Evidence Manifest

## Current baseline

- Current public prerelease: `v0.9.0rc3`
- Current main development version: `0.9.0rc4.dev0`
- PyPI/TestPyPI: not published / not uploaded
- Open-source-solver-first strategy: yes

## Evidence categories

- Unit tests: model, parser, validator, adapter, workflow, API, and utility tests.
- Contract tests: CLI, schema, adapter support matrix, workflow preview, package
  metadata, and release engineering checks.
- Documented CLI examples: offline examples in `examples/README.md` and
  `examples/examples_manifest.json`.
- Adapter evidence fixtures: stable fragments under
  `tests/fixtures/adapter_golden/`.
- Workflow preview fixtures: stable keys under
  `tests/fixtures/workflow_preview/`.
- Schema compatibility tests: `tests/test_schema_compatibility_policy.py` and
  schema contract tests.
- Failure-mode regression tests: `tests/test_failure_mode_regression.py` and
  adapter failure-mode checks.
- `scripts/smoke_release.sh`: clean install, test, build, dist filename, and CLI
  smoke verification.
- Wheel install smoke: optional local wheel installation check.
- Build artifacts: local wheel/sdist generated by `python -m build`.
- `make check`: deterministic tests, parser benchmarks, semantic benchmark,
  mock LLM benchmark, workflow benchmark, docs checks, CLI checks, and artifact
  contract checks.

## What this evidence supports

- Local parsing behavior.
- Schema validation behavior.
- CLI availability.
- Adapter local artifact preview.
- Workflow planning preview.
- Packaging sanity.
- No-network default examples.
- No external solver/LLM default path.
- Open-source-solver-first release engineering.

## What this evidence does not support

- No production-grade physical validation.
- No formal convergence proof.
- No solver-backed correctness claim by default.
- No proprietary commercial solver validation.
- No PyPI publication claim.
